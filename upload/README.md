# 실시간 네트워크 트래픽 송/수신 파이프라인

머신러닝 기반 침입 탐지를 위한 실시간 트래픽 수집·전송 파이프라인입니다.

공격 트래픽을 캡처하여 CICIDS2017 형식으로 변환한 뒤, 예측 서버까지 자동으로 전달합니다.

## 전체 흐름

A (캡처+변환)  →  B (정규화)  →  C (수신/예측)

트래픽 캡처부터 예측 서버 도착까지 사람 개입 없이 자동으로 처리됩니다.

## 구성

### A_Upload2.py — 캡처 및 전송

- `tcpdump`로 10초 단위 PCAP 파일 생성 (파일명: `cap_년월일_시간.pcap`)
- `CICFlowMeter`로 PCAP → CSV 변환
- 변환된 CSV를 B(정규화 서버)로 전송
- 전송 완료된 파일은 자동 삭제 (디스크 누적 방지)
- 실행 시 이전 PCAP/CSV 및 잔여 tcpdump 프로세스 정리 후 시작

### B_normalize.py — 정규화 및 전달

- A로부터 받은 CSV를 `received/` 디렉터리에 저장
- 팀 정규화 스크립트를 실행하여 CICIDS2017 형식으로 변환
- 정규화된 CSV를 C(예측 서버)로 전송
- 전송 완료 후 중간 파일 자동 삭제

### C_received.py — 수신 및 예측

- 6000 포트를 열고 대기
- B가 보낸 정규화 CSV를 `received/` 디렉터리에 저장
- 예측 모델 연동 지점 확보 (추후 모델 담당 파트에서 연결)

## 실행 순서

수신 측부터 역순으로 실행합니다.

```bash
# 1. C (예측 서버)
python C_received.py

# 2. B (정규화 서버)
python B_normalize.py

# 3. A (캡처 서버)
sudo python3 A_Upload2.py
```

## 설정

각 스크립트 상단의 `CONFIG` 영역에서 환경에 맞게 수정합니다.

| 항목 | 설명 |
|------|------|
| `INTERFACE` | A의 캡처 인터페이스 (`ip a`로 확인) |
| `UPLOAD_URL` | A → B 전송 주소 (B의 Tailscale IP:5000) |
| `PREDICT_URL` | B → C 전송 주소 (C의 Tailscale IP:6000) |
| `NORMALIZE_SCRIPT` | 정규화 스크립트 파일명 |

## 특징

- **무손실 캡처**: tcpdump 연속 실행으로 패킷 누락 없음
- **자동 정리**: 전송 완료 파일 자동 삭제로 디스크 누적 방지
- **장애 대응**: 전송 실패 시 파일 보존 및 재시도, 개별 처리 실패가 전체를 중단시키지 않음
- **로깅**: 각 단계별 처리 내역을 콘솔 및 로그 파일에 기록
